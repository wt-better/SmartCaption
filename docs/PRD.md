# SmartCaption 智能字幕Web应用 PRD

## 1. 产品概述

### 1.1 产品定位
SmartCaption 是一款基于 Web 的智能字幕生成应用，支持从音视频文件自动生成字幕，并提供多语言翻译能力。产品定位为"让每一帧画面都能美妙地表达"，为用户提供便捷、高效、智能的字幕处理解决方案。

### 1.2 目标用户
- 视频创作者：需要为视频添加字幕
- 翻译工作者：需要批量处理字幕翻译
- 教育工作者：制作双语教学视频
- 内容运营：处理多语言内容分发

### 1.3 核心价值
- **智能化**：自动语言检测、智能分段、AI翻译
- **可视化**：全流程可视化，实时进度追踪
- **灵活性**：多模型支持、参数可配置
- **易用性**：Web界面，无需安装，跨平台使用

## 2. 核心功能模块

### 2.1 文件上传模块

#### 2.1.1 文件上传
- **支持格式**
  - 视频：MP4, MKV, AVI, MOV, WebM, FLV, WMV
  - 音频：MP3, WAV, FLAC, AAC, OGG, M4A
  - 字幕：SRT, VTT, ASS, SSA, TXT（用于翻译场景）

- **上传特性**
  - 拖拽上传 + 点击选择
  - 单文件上传
  - 大文件分片上传（>100MB）
  - 上传进度实时显示
  - 文件类型自动识别

#### 2.1.2 文件预览
- 视频文件：提供视频预览播放器
- 音频文件：提供音频波形预览 + 播放控制
- 字幕文件：提供文本预览
- 文件信息展示：大小、时长、格式、编码等

#### 2.1.3 文件操作
- 删除重新上传

### 2.2 字幕生成配置模块

#### 2.2.1 ASR模型配置

| 模型 | 参数量 | 特点 | 适用场景 |
|------|--------|------|----------|
| Qwen/Qwen3-ASR-1.7B | 1.7B | 中文效果优秀，速度快 | 中文视频、实时处理 |
| Qwen/Qwen3-ASR-0.6B | 0.6B | 轻量级，资源占用低 | 低配置设备、快速预览 |
| openai/whisper-large-v3 | 1.5B | 多语言支持好，精度高 | 多语言场景 |
| openai/whisper-large-v2 | 1.5B | 稳定可靠 | 通用场景 |

**配置项**
- 模型选择（默认：Qwen/Qwen3-ASR-1.7B）
- 设备选择：CPU / GPU（自动检测）
- 语言提示（可选，提高识别准确率）

#### 2.2.2 语言配置

**语言检测流程**
```
用户上传文件 → 自动语言检测 → 显示检测结果 → 用户确认/修改
```

**语言选择规则**
- 源语言：自动检测，用户可修改
- 目标语言：最多选择2种
- 语言选择联动翻译模块

**支持语言参考**
| 语言 | 代码 | 语言 | 代码 |

| 德语 | de | 西班牙语 | es |
| 俄语 | ru | 葡萄牙语 | pt |
| 意大利语 | it | 阿拉伯语 | ar |

#### 2.2.3 翻译配置（条件触发）

**触发条件**：目标语言 ≠ 源语言

**翻译模型管理**
- 模型列表管理（增删改查）
- API 配置管理（Base URL、API Key、模型名称）

**翻译参数**
- 翻译模型选择
- 初始提示词（专业术语引导）
- 并发数设置

**基础样式**
- 字幕位置：底部/顶部/自定义
- 字体选择：系统字体列表
- 字体大小：小/中/大/自定义

#### 2.3.2 步骤详解

**Step 1: 文件预处理**
- 检测文件时长
- 判断是否需要分割（阈值：5分钟）
- 提取音频轨道（如果是视频）
- 音频格式标准化

**Step 2: 大文件分割（条件执行）**
- 触发条件：文件时长 > 5分钟
- 分割策略：
  - 按静音段分割（优先）
  - 固定时长分割（5分钟/段）
  - 分割点平滑过渡
- 生成分割信息记录

**Step 3: ASR处理**
- 模型加载
- 分段处理（分割场景）
- 子任务串行执行
- 生成时间轴对齐的字幕

**Step 4: 翻译处理（条件执行）**
- 触发条件：目标语言 ≠ 源语言
- 翻译模型调用
- 分段翻译（分割场景）
- 翻译结果合并

**Step 5: 字幕合并**
- 触发条件：进行了文件分割
- 时间轴校准
- 字幕片段拼接
- 格式统一

**Step 6: 视频合成**
- FFmpeg字幕嵌入
- 硬字幕/软字幕选择
- 输出文件生成

**Step 7: 完成处理**
- 结果预览
- 文件下载
- 中间文件管理

### 3.3 数据模型

#### 3.3.1 任务模型 (Task)
```python
class Task:
    id: str                    # 任务ID
    user_id: str               # 用户ID
    status: TaskStatus         # 状态: pending/running/completed/failed
    created_at: datetime       # 创建时间
    updated_at: datetime       # 更新时间
    
    # 文件信息
    source_file: FileInfo      # 源文件信息
    output_file: FileInfo      # 输出文件信息
    
    # 配置信息
    config: TaskConfig         # 任务配置
    
    # 执行信息
    steps: List[Step]          # 执行步骤列表
    current_step: int          # 当前步骤
    current_step_status: str   # 当前步骤状态
    
    # 结果信息
    result: TaskResult         # 执行结果
    error: ErrorInfo           # 错误信息
```

#### 3.3.2 步骤模型 (Step)
```python
class Step:
    id: str                    # 步骤 ID
    task_id: str               # 所属任务 ID
    name: str                  # 步骤名称
    type: StepType             # 步骤类型
    status: StepStatus         # 状态
    order: int                 # 执行顺序
    
    # 子任务（分割场景）
    sub_steps: List[SubStep]   # 子步骤列表
    current_sub_step: int      # 当前子步骤
    
    # 执行信息
    started_at: datetime       # 开始时间
    finished_at: datetime      # 结束时间
    
    # 日志
    logs: List[LogEntry]       # 日志列表
```

#### 3.3.3 配置模型 (TaskConfig)
```python
class TaskConfig:
    # ASR配置
    asr_model: str             # ASR模型
    asr_device: str            # 推理设备
    asr_params: dict           # ASR参数
    
    # 语言配置
    source_language: str       # 源语言
    target_languages: List[str] # 目标语言列表
    
    # 翻译配置（如果目标语言≠源语言则启用）
    translation_model: str     # 翻译模型
    translation_prompt: str    # 翻译提示词
    concurrent_tasks: int      # 并发任务数
    
    # 字幕配置
    subtitle_style: SubtitleStyle  # 字幕样式
    
    # 处理配置
    split_threshold: int       # 分割阈值 (秒)
    max_concurrent: int        # 最大并发数
```

## 4. API接口设计

### 4.1 文件接口

```
POST   /api/v1/files/upload          # 文件上传
GET    /api/v1/files/{file_id}       # 获取文件信息
DELETE /api/v1/files/{file_id}       # 删除文件
GET    /api/v1/files/{file_id}/preview # 获取预览信息
```

### 4.2 任务接口

```
POST   /api/v1/tasks                 # 创建任务
GET    /api/v1/tasks/{task_id}       # 获取任务详情
DELETE /api/v1/tasks/{task_id}       # 删除任务
POST   /api/v1/tasks/{task_id}/retry # 重试任务
GET    /api/v1/tasks/{task_id}/download # 下载结果
```

### 4.3 配置接口

```
GET    /api/v1/config/models         # 获取可用模型列表（ASR + 翻译）
GET    /api/v1/config/languages      # 获取支持语言列表
GET    /api/v1/config/translation-models # 获取翻译模型列表
POST   /api/v1/config/translation-models # 添加翻译模型
PUT    /api/v1/config/translation-models/{id} # 更新翻译模型
DELETE /api/v1/config/translation-models/{id} # 删除翻译模型
```

### 4.4 WebSocket 接口

```
WS     /api/v1/ws/tasks/{task_id}    # 任务进度推送

  }
}
```

## 6. 跨平台支持

### 6.1 操作系统支持

| 操作系统 | 版本要求 | 支持架构 | 特性说明 |
|----------|----------|----------|----------|
| Windows | 10/11 | x64, ARM64 | 完整功能支持，GPU 加速（CUDA） |
| macOS | 12.0+ | Intel, Apple Silicon | 完整功能支持，Core ML 加速（M 系列） |
| Linux | Ubuntu 20.04+, Debian 11+ | x64, ARM64 | 完整功能支持，GPU 加速（CUDA） |

### 6.2 跨平台实现策略

**前端跨平台**
- Web 应用：浏览器即平台，无需区分操作系统

**后端跨平台**
- Python 跨平台框架（FastAPI）
- 统一 API 接口，操作系统差异内部屏蔽
- FFmpeg 跨平台二进制包管理

**文件处理跨平台**
- 路径处理：使用跨平台路径库
- 文件编码：统一使用 UTF-8
- 换行符：自动处理 CRLF/LF 差异
